它能捕捉什么

只记录重要的，绝不多此一举。

四项轻量信号如实勾勒出一天的工作 — 不抓取屏幕内容，也不记录任何人的输入。

应用 & 窗口时长

按应用和窗口统计的空闲感知活跃时长。空闲间隙以斜纹标出，不计入时长。

按键量

仅统计次数 — 用以衡量打字活跃度。从不捕捉或存储实际按键内容。

定时屏幕截图

按设定间隔自动截图，压缩至 50 KB 以内，并设有自动删除的保留窗口。

浏览器页面 & 时长

一个轻巧的 Manifest V3 扩展上报活跃页面和时长 — 仅发送给本地应用。

本地优先存储

所有数据都落入单个本地 SQLite 文件。向所有者控制台的同步是可选的、单向的。

菜单栏状态

安静的菜单栏指示器显示追踪中、空闲或已暂停 — 并让你随时开始、停止或打开应用。

隐私源于设计

绝不记录按键内容

键盘信号是每个时段的按键次数统计。按键本身从不被读取或存储。

开源

不想付费？自己来托管。

BiBoTracking 是开源的 — Go 后端、网页控制台、macOS 应用和 Chrome 扩展全都任你阅读、审计并在自己的机器上运行，永久免费。这里的 托管服务只是更省心的选择。

自托管整套系统

一个 Go 二进制文件即可提供 API 和控制台，由 Postgres 支撑。你的基础设施，你的规则。

一切尽在眼底

阅读源码，精确核实捕捉了什么 — 以及什么永远不会离开设备。

绝无锁定

你的数据存在自己的 SQLite 和 Postgres 中。随时导出、迁移、保留。
