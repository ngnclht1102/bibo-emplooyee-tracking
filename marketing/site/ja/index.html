記録する内容

大切なものすべてを。不要なものは一切なし。

4つの軽量なシグナルが、画面のコンテンツを取得したり入力内容を記録したりすることなく、一日の業務の正直な姿を映し出します。

アプリ&ウィンドウの時間

アプリとウィンドウごとの、アイドルを考慮したアクティブ時間。アイドルの空白はハッチングで示され、カウントされません。

キーストローク量

回数のみ — タイピング活動の指標です。実際のキーが取得・保存されることはありません。

定期的なスクリーンショット

設定した間隔で自動的にキャプチャし、50 KB未満に圧縮。自動削除の保持期間付きです。

ブラウザのページ&時間

軽量なManifest V3拡張機能が、アクティブなページと時間を — ローカルアプリにのみ報告します。

ローカルファーストのストレージ

すべてが単一のローカルSQLiteファイルに収まります。オーナーダッシュボードへの同期は任意かつ一方向です。

メニューバーのステータス

控えめなメニューバーのインジケーターが、追跡中・アイドル・一時停止を表示 — 開始・停止やアプリの起動も行えます。

使い方

3ステップですぐに始められます。

1

macOSアプリをインストール

従業員がBiBoTrackingをインストールし、アクセシビリティ、入力監視、画面収録の権限を付与します。権限が付与された瞬間にステータスが自動更新されます。

2

Chrome拡張機能を追加

任意です。Manifest V3拡張機能がローカルアプリを自動検出し、ブラウザのページを 127.0.0.1 に報告します — クラウドには一切送信しません。 Chrome Web Storeで入手 →

3

オーナーは集約を確認

アクティビティは一方向にダッシュボードへ同期され、オーナーは従業員ごとの時間、キーストローク、ブラウザ履歴、スクリーンショットを確認できます。

プライバシー・バイ・デザイン

人を尊重するモニタリング。

透明性は設定項目ではなく — 製品そのものの作り方です。

キーは決して記録されません

キーボードのシグナルは、区間ごとの押下回数です。キーそのものが読み取られたり保存されたりすることはありません。

ローカルファーストのストレージ

すべてのアクティビティはローカルのSQLiteファイルに収まります。ダッシュボードへの同期は任意かつ一方向で、冪等です。

ドメインのみのオプション

ブラウザ追跡では、完全なURLの代わりにドメインだけを保存することもできます — あなたのポリシー、あなたの判断で。

あなたが制御する保持期間

スクリーンショットは設定した期間を過ぎると自動削除され、キャプチャポリシー全体は組織によって一元管理されます。

オープンソース

支払いたくない？ ご自身でホストを。

BiBoTrackingはオープンソースです — Goのバックエンド、Webダッシュボード、macOSアプリ、Chrome 拡張機能はすべて、ご自身のマシンで読み・監査し・実行できます。永久に無料です。 ここで提供するホスティングサービスは、あくまで便利な選択肢にすぎません。

スタック全体をセルフホスト

1つのGoバイナリがAPIとダッシュボードを提供し、Postgresを基盤とします。あなたのインフラ、あなたのルールで。

すべてを検証

ソースを読み、何が記録され — 何が決してデバイスから出ないのかを、正確に確認できます。

ロックインなし

あなたのデータは、あなた自身のSQLiteとPostgresに保存されます。いつでもエクスポート・移行・保持が可能です。
