# 110 — QA: i18n across web, desktop, and marketing (native review)

- **Phase:** 9
- **Type:** Interactive QA
- **Status:** Blocked
- **Blocked by:** 107, 108, 109

## Goal
Confirm the localization is correct, complete, and native-quality across all three
surfaces and all 6 non-English locales — a human runs each app/page per language.

## Checklist (per locale: zh-Hans, ja, vi, id, fr, es)
- **Completeness:** no English leakage; no raw keys (`auth.signIn.title`) visible; no
  empty strings. Grep catalogs for missing keys vs the `en` source.
- **Native quality:** a fluent speaker reviews each surface for meaning, tone, and
  grammar — not just literal correctness. Personas (employee/kid, team/family) read
  naturally. Privacy/permission copy is exact (no softened or overstated meaning).
- **Formatting:** numbers, dates, relative "last seen", and durations format per
  locale via `Intl`. Plurals correct (0/1/many where the language distinguishes).
- **Layout:** longer FR/ES strings and CJK don't clip buttons, the signup rail, table
  headers, tray menu, or the desktop sidebar. CJK line-height looks right.
- **Switching & persistence:** changing language updates everything live; persists
  across web reload and desktop restart; native desktop strings (tray/notifications)
  follow the choice.
- **Marketing SEO:** validate `hreflang`/`x-default`, per-locale title/description/OG,
  JSON-LD, and `sitemap.xml`; Lighthouse SEO per locale page.
- **Fallback:** an unset/partial locale falls back to English without crashing.

## Acceptance criteria
- [ ] Each locale signed off by a fluent reviewer on web + desktop + marketing.
- [ ] No missing keys / English leakage / layout breakage in any locale.
- [ ] Formatting + plurals correct per locale.
- [ ] Marketing international SEO validated.

## Notes
- Track sign-offs in a small matrix (locale × surface) attached to this ticket.
- File follow-up tickets for any locale that needs a professional re-translation pass.
